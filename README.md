# 시험정보 카메라 (웹앱 / 사파리에서 실행)

아이폰·안드로이드 모두 **주소(링크)만 열면** 바로 실행되는 웹앱입니다.
설치·서명·7일 제약이 없습니다. 카메라를 켜면 화면 우측 하단에 **시험정보**가 뜨고,
촬영하면 그 내용이 **사진에 새겨져** 저장됩니다. 접수번호별로 관리되어 이어찍기도 됩니다.

## 들어있는 파일 (이게 전부예요)
```
index.html
app.js
manifest.webmanifest
sw.js
icon-180.png
icon-192.png
icon-512.png
```

## 무료로 인터넷에 올리기 (GitHub Pages — 카메라는 HTTPS에서만 켜집니다)
1. GitHub에서 **새 저장소(New repository)** 를 하나 만듭니다. (이름 예: `test-cam`, Public)
   - 안드로이드 빌드용 `csi` 저장소와 섞이지 않게 **새 저장소**를 권장합니다.
2. **Add file → Upload files** 로 위 7개 파일을 **모두** 올리고 **Commit changes**.
   - 파일들이 저장소 **맨 위(루트)** 에 있어야 합니다. (`index.html`이 바로 보여야 정상)
3. 저장소 상단 **Settings → 왼쪽 메뉴 Pages** 로 이동.
4. **Build and deployment → Source: Deploy from a branch** 선택,
   Branch를 **main / (root)** 으로 놓고 **Save**.
5. 잠시(1~2분) 뒤 같은 Pages 화면 위쪽에 **주소**가 뜹니다:
   `https://<아이디>.github.io/test-cam/`
6. 이 주소를 **아이폰 사파리(또는 안드로이드 크롬)** 에서 엽니다.

## 아이폰에서 쓰기
1. 사파리로 위 주소 열기 → 카메라 촬영 시 **카메라 접근 허용**을 한 번 물어봄 → 허용
2. (권장) 사파리 하단 **공유 버튼 → ‘홈 화면에 추가’** 를 누르면
   - 앱 아이콘이 생겨 전체화면으로 열리고,
   - 저장한 접수건/사진 목록이 안정적으로 유지됩니다.
   - ※ 홈 화면에 추가하지 않으면, 사파리가 오래 안 쓴 사이트의 저장 데이터를 지울 수 있어요(약 7일). 실제 사진은 아래 ‘저장/공유’로 사진 앱에 넣으면 안전합니다.

## 사용 순서
1. **＋ 새 접수건**
2. (편함) **📷 신청서 스캔해서 자동입력** → 신청서 촬영 → 접수번호·CSI·공사명·시료명·비고 자동 채움 → **확인·수정**
   - 직접 입력해도 됩니다. 스캔은 인터넷이 필요하고, 손글씨는 틀릴 수 있어요.
3. **저장하고 촬영 시작**
4. **시험전/중/후**, **시험항목** 선택(없으면 ＋로 추가) → 가운데 빨간 버튼으로 촬영
5. 촬영 후 **‘사진 앱에 저장 / 공유’** 로 폰 사진 앱에도 저장 (앱 안 갤러리엔 자동 저장됨)
6. 나중에 첫 화면에서 **해당 접수건을 눌러** 이어찍기 / **사진 보기**에서 전체 저장

## 참고
- **카메라는 HTTPS 주소에서만** 켜집니다. GitHub Pages 주소(https://...)면 문제없어요. (파일을 그냥 더블클릭해서 여는 `file://` 방식은 카메라가 안 켜집니다.)
- 앱 안 갤러리는 이 기기(브라우저) 안에 저장됩니다. 다른 기기와 자동 동기화되진 않아요. 사진을 확실히 남기려면 ‘저장/공유’로 사진 앱/드라이브에 보관하세요.
- 아이폰·안드로이드 같은 주소를 쓰면 둘 다에서 동작합니다.
